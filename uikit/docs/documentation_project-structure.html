项目结构

深入挖掘UIkit的核心部分，了解关于它的一切。

GitHub

UIkit被托管在 GitHub 上，基于 MIT 许可协议。 欢迎使用它应用于个人或商业项目。想要获取 UIKit 所有的源文件，只需克隆 GitHub 上的资源库，或者直接下载最新的版本。

下载源代码

文件结构

UIkit 基于LESS创建，LESS是一款CSS预处理器，将 CSS 赋予了动态语言的特性，如变量，继承，运算，函数等。它将写入LESS中的代码编译成为CSS。 UIkit主要的文件结构如下。

文件夹 概述
/addons/src 包含所有附加组件相关的LESS和JavaScript文件。
/addons/tests 包含所有附加组件的测试文件。
/dist 包含所有已编译的及精缩CSS和JavaScript文件，以及字体。
/docs 包含你现在正在看的所有UIKit说明文档，。
/src/less 包含所有LESS组件文件。
/src/js 包含所有JavaScript组件文件。
/src/fonts 包含UIKit中使用的字体。
/src/tests 包含UIKit所有组件的测试文件。
/themes 在子文件夹中包含的所有额外提供的主题文件。
/vendor 包含UIkit所使用的jQuery和其他的外部库。

自动化预处理

我们利用 Grunt（一个基于 Node.js 的自动化的项目构建工具），自动将LESS文件编译为CSS，它安装在你的工作目录中并监控所有的操作。每当你保存更改后的源文件，Grunt 将自动把所有文件编译成一个单独的CSS文件。

首先，你需要安装Node，并设置 grunt.cli 作为一个全局的安装，然后切换到UIkit目录，并安装Node dependencies。

npm install -g grunt-cli

cd uikit
npm install

现在你可以运行 Grunt 建立和修改版本。UIkit的内置版本被放在 /dist 目录中。

grunt

你还可以设置 Grunt 监控你的工作目录，这样它就能随时自动预处理你所保存的文件。

grunt watch

加载新的主题和风格样式到定制器。

grunt indexthemes

测试

UIkit为每个组件提供测试文件。其中每个页面都包含了组件的测试示例，并为你提供了一个关于盒子所支持的所有可能性的总览。

代替预编译的CSS，我们使用客户端JavaScript来转化LESS文件，因此当你正在制作你自己的主题，或者修改现有的主题时，你能在浏览器中看到即时的变化。你能在测试中切换所有的组件和主题，甚至切换到RTL模式（从右到左）。
